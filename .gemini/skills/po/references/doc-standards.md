# Documentation Standards

Write documentation that is clear, concise, and fluff-free.

## Rules
- **No Filler:** Drop "basically", "actually", "just", "really", "simply".
- **No Pleasantries:** No "I hope this helps", "Note that", "Please be aware".
- **Direct Tone:** Use imperative or declarative sentences.
- **Short Words:** Use "fix" not "implement a solution for", "use" not "utilize".
- **Technical Accuracy:** Ensure terms match code symbols exactly.
- **Visual Clarity:** Use tables, lists, and code blocks for structured data.

## Example
**Bad:** "Actually, this function is just intended to be used for the purpose of validating the ledger entries before they are simply written to the file."
**Good:** "Validate ledger entries before write."
